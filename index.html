1. Mise en place - Environnement

Étape 1 : Téléchargement des logiciels

  1. Visual Studio Code
  2. GitHub Desktop
  3. Git
  4. Node.js

⚠️ Une fois tous les logiciels installés, redémarrez votre ordinateur. ⚠️

Étape 2 : Mise en place - GitHub

Tout d'abord, il va falloir forker le projet en se rendant ici après avoir préalablement créé un projet vierge. Votre fork devra être public.

  1. Créez ensuite un dossier vide à l'emplacement de votre choix sur votre PC.

  2. Ouvrez GitHub Desktop et clonez votre repository fork.

  3. Il vous faudra ensuite vous rendre dans "File" (en haut à gauche du logiciel), puis dans "Clone Repository" (voir Premier Screenshot).

  4. Choisissez ensuite le dossier que vous venez de créer, et sélectionnez dans le menu déroulant au-dessus votre projet que vous avez forké avec le CentralCorp Launcher. Le clonage commencera alors.

Étape 3 : Configuration globale

Une fois le clonage terminé, sélectionnez "Open in Visual Studio Code", ce qui ouvrira le dossier contenant la base du launcher.

Vous allez maintenant vous rendre dans le fichier package.json et configurer les éléments suivants :

  • name : Le nom de votre launcher
  • productName : Le nom qui sera affiché sur le raccourci du bureau et dans le gestionnaire d'applications
  • version : La version de votre launcher (PAS votre version de Minecraft)*
  • description : La description qui sera notamment affichée en passant la souris sur le raccourci
  • author : Votre pseudonyme (Merci de laisser également Luuxius/Riptiaz pour les crédits :heart: (supprimer = ban))
  • settings : Le lien vers votre panel

Une fois cela fait, ouvrez un terminal dans votre éditeur de code et tapez la commande :

npm install

La version ne doit pas être choisie au hasard. Elle sera utilisée pour la compilation du launcher. Rendez-vous à la partie 4 pour plus d'informations.

Étape 4 : Mise en place du panel - 1

1. Téléchargement des fichiers nécessaires

Tout d'abord, vous avez deux options :

Option 1

  1. Rendez-vous sur ce lien et cliquez sur le gros bouton vert "Code", puis sur "Download ZIP".
  2. Extrayez ensuite l'archive sur votre bureau ou dans un dossier de votre choix.

Et c'est tout pour cette partie ! (Eh oui, c'était rapide :p)

Option 2

Téléchargez le fichier Panel.zip fourni dans ce tutoriel (plus rapide).

2. Mise en place du panel sur un serveur web

Ce qu'il vous faudra pour continuer :

  • Un serveur web fonctionnel sous PHP (parfois payant).
  • Une base de données vierge (obligatoire).

Si vous n'avez pas d'hébergement, vous pouvez utiliser celui-ci : https://hwhost.fr/, qui est normalement compatible et fournit tout ce dont le panel a besoin, avec un abonnement Gratuit.

Étapes à suivre :

  1. Connectez-vous au serveur FTP/SFTP de votre serveur web et téléchargez-y tous les fichiers du panel que vous avez précédemment téléchargés.

    Note

    • Vous ne pourrez pas accéder au panel si vous l'hébergez sur le même serveur que votre site Azuriom. Il vous faudra obligatoirement un sous-domaine ou un autre hébergement.
    • Certains hébergeurs proposent un web FTP. Dans ce cas, uploadez les fichiers dans le dossier httpdocs.

  2. Une fois tous les fichiers du panel uploadés sur votre serveur FTP/SFTP, vous pouvez accéder à votre panel à l'adresse de votre serveur web.

  3. Configurez ensuite la page qui apparaît avec les identifiants de votre base de données.

  4. Une fois la base de données renseignée, et si celle-ci est bien vide, vous pourrez créer un compte. Ne perdez pas vos identifiants, ce sont les seuls qui vous permettront d'accéder au panel.

5. Configuration launcher depuis le Panel de gestion

1. Configuration

Avant de commencer ce tutoriel, nous partons du principe que vous avez suivi et complété correctement le tutoriel sur la mise en place du panel.

Il va donc falloir que vous vous rendiez sur votre panel et que vous vous y connectiez si cela n'est pas déjà fait. Ensuite, le fonctionnement est assez simple :

Il existe 3 types de configuration :

  1. Les configurations dites "textuelles" : Vous devrez écrire ou coller un texte qui sera ensuite retranscrit dans le launcher.

  2. Les configurations dites "sélectives" : Vous pouvez cocher ou décocher les options disponibles.

  3. Les configurations dites "déroulantes" : Un menu déroulant vous permettra de choisir parmi les options disponibles.

Nous n'allons pas vous présenter toutes les options disponibles sur le panel, car elles ne seraient jamais à jour étant donné que nous mettons régulièrement le panel à jour. Cependant, je vais vous présenter les plus importantes.

Configuration des options principales

  1. URL du site Azuriom :
  2. Rendez-vous dans la section "URL du site Azuriom" et indiquez l'URL de votre site, sans ajouter de "/" à la fin (sinon cela ne fonctionnera pas).

exemple : 'https://conflictura.eu' et non 'https://conflictura.eu/

  1. Type de loader :
  2. Sélectionnez le loader que vous souhaitez utiliser. Dans cet exemple, nous choisirons Forge, car il est le plus répandu. Rendez-vous sur le site de Forge, sélectionnez votre version et copiez la version de build que vous souhaitez. Cela correspond à la suite de caractères commençant par la version de Minecraft, par exemple 1.20.1 - 47.1.44.

1. Hébergement des fichiers

Comme un launcher custom sert très souvent à faire télécharger vos fichiers à d'autres personnes (mods, configurations, etc.), il est nécessaire de configurer et renseigner le lien de vos fichiers dans le launcher.

Étapes à suivre :

  1. Accédez au dossier de votre panel, puis allez dans le dossier "data".
  2. Créez un dossier nommé files.
  3. Glissez-y tous vos fichiers en les organisant de la même manière que dans votre répertoire de jeu (par exemple, les mods dans un dossier nommé mods, les configurations dans un dossier nommé config, etc.).

Vous pourrez modifier tous les fichiers de votre jeu depuis ce répertoire, et ces modifications seront automatiquement téléchargées par tous vos joueurs.

6. Compilation du launcher

Nous y sommes ! Vous allez enfin obtenir votre launcher au format .exe à fournir à tous vos joueurs. Mais avant cela, voici les étapes à suivre :

1. Configurer le fichier package.json :

  • Retournez dans Visual Studio Code et ouvrez le fichier package.json.
  • Allez tout en bas, dans la ligne "url" sous "repository".
  • Après git+, collez le lien de votre projet. Cette étape est importante.
  • Gardez en tête la version que vous avez renseignée plus haut.

2. Créer une nouvelle version sur GitHub :

  • Rendez-vous sur GitHub, dans l'onglet "Releases", et cliquez sur "Draft a new release".
  • Dans "Choose a tag", notez EXACTEMENT ce qui est écrit sur la ligne version, puis cliquez sur "Create new tag".
  • Renseignez les autres champs comme vous le souhaitez, puis cliquez sur "Save draft".

3. Modifier les paramètres du projet :

  • Allez dans les paramètres de votre projet sur GitHub, puis dans "Actions" > "General".
  • Faites défiler jusqu'à "Workflow permissions", réglez la valeur sur le premier point, et enregistrez.

4. Mettre à jour et pousser les changements :

  • Ouvrez un terminal dans votre éditeur de code et tapez npm i pour mettre à jour toutes les dépendances.
  • Ensuite, ouvrez GitHub Desktop. Vous devriez voir une interface similaire à celle du screenshot n°1.
  • Écrivez quelque chose dans "Summary", puis cliquez sur "Commit to master".
  • Cliquez sur le bouton bleu "Push origin" au centre de l'écran.

5. Vérifier le processus de compilation :

  • Rendez-vous sur votre projet GitHub, puis dans "Actions". Vous devriez voir un point orange clignotant.
  • Si tout se passe bien, après quelques minutes, vous pourrez récupérer votre launcher dans tous les formats exécutables.
